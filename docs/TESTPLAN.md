# COBOL Student Account System Test Plan

This test plan covers the current business logic and observable implementation behavior of the COBOL student account application. It is intended for stakeholder validation now and can later be used as the basis for Node.js unit and integration tests during modernization.

## Scope

- Menu-based account operations in `MainProgram`
- Account business rules in `Operations`
- Balance storage and retrieval behavior in `DataProgram`
- Current implementation constraints that affect expected behavior

## Notes for Stakeholder Review

- The current COBOL implementation has clear business rules for viewing balance, crediting funds, debiting funds, and rejecting overdrafts.
- The current implementation does not explicitly define business rules for zero amounts, negative amounts, or non-numeric input. Those cases are included in this plan so stakeholders can decide whether the Node.js application should preserve current behavior or enforce stricter validation.
- `Actual Result`, `Status (Pass/Fail)`, and `Comments` are intentionally left blank for execution during validation sessions.
